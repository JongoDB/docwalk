Troubleshooting¶

Verbatim‑Studio Troubleshooting Guide¶

For end‑users – quick, friendly help when things go sideways.

1. Quick‑Start¶

Step What to do Why it matters
1️⃣ Install pip install verbatim-studio (Python) or npm i verbatim-studio (Node) Gets the latest stable binaries.
2️⃣ Verify verbatim-studio --version Confirms the tool is reachable.
3️⃣ Configure Edit ~/.verbatim-studio/config.yaml (or config.json) Sets storage paths, auth tokens, etc.
4️⃣ Run verbatim-studio process myfile.txt Starts a document‑processing job.

Tip – If you’re new, run verbatim-studio help to see all commands and options.

2. Common Issues & Fixes¶

Below are the most frequently reported errors. For each, we explain what triggers it and give a step‑by‑step fix.

⚠️ StorageError – Base storage error.
Cause – A generic failure when accessing the storage backend.
Fix – Check the detailed sub‑error (see below) or try a fresh connection.

2.1 StorageUnavailableError¶

What it means
The storage location (e.g., a cloud bucket or a local folder) cannot be reached.
Typical symptoms
“Unable to connect to storage”
 Timeout or DNS resolution failures

Steps to fix

  1. Ping the storage host
    ping <storage-host>
  2. Check network connectivity
    curl -I https://<storage-host>
  3. Verify that the storage service is running (e.g., S3, GCS, local NFS).
  4. If using a VPN or proxy, ensure it’s active and that the required ports are open.

✅ If the host is reachable, try restarting Verbatim‑Studio or re‑authenticating.

2.2 StorageConnectionError¶

What it means
The client could not establish a connection to the storage backend.
Typical symptoms
“Connection refused”
 “Failed to open socket”

Steps to fix

  1. Confirm the endpoint URL in config.yaml.
  2. Test the connection manually
    curl -v https://<storage-endpoint>
  3. Check firewall rules – ensure outbound traffic to the endpoint is allowed.
  4. If using TLS, verify certificates (openssl s_client -connect <host>:443).

⚠️ If you’re behind a corporate proxy, set HTTPS_PROXY and HTTP_PROXY environment variables.

2.3 StorageAuthError¶

What it means
Authentication to the storage service failed or the token has expired.
Typical symptoms
“401 Unauthorized”
 “Invalid credentials”

Steps to fix

  1. Refresh your credentials
  2. For AWS: aws configure
  3. For GCP: gcloud auth login
  4. For local services: re‑enter username/password in config.yaml
  5. Verify the token’s expiry (date -d @<epoch>).
  6. Check the scope/permissions – the account must have read/write rights to the bucket/folder.

✅ After updating credentials, restart the process.

2.4 StorageNotFoundError¶

What it means
The requested file or directory does not exist in the storage location.
Typical symptoms
“File not found”
 “404 Not Found”

Steps to fix

  1. Double‑check the path in your command or config.
  2. List the contents of the target folder to confirm the file’s presence.
    verbatim-studio ls <bucket-or-folder>
  3. If the file is missing, upload it or correct the path.

⚠️ Remember that paths are case‑sensitive on most storage backends.

2.5 StoragePermissionError¶

What it means
The user/account lacks the necessary permissions to read or write.
Typical symptoms
“403 Forbidden”
 “Permission denied”

Steps to fix

  1. Check IAM policies (AWS IAM, GCP IAM, etc.) for the bucket/folder.
  2. Grant the required role (e.g., storage.objectAdmin).
  3. If using a local filesystem, ensure the file permissions allow the user running Verbatim‑Studio.
    ls -l /path/to/file
chmod 664 /path/to/file

✅ After updating permissions, retry the operation.

2.6 ProcessingCancelledError¶

What it means
The document processing job was cancelled by the user or by an external trigger.
Typical symptoms
“Processing cancelled”
 Job stops mid‑way

Steps to fix

  1. Check if you pressed Ctrl+C or sent a SIGTERM – if so, that’s expected.
  2. If cancellation is unexpected, look for a cancel flag in your config or a background job manager that might be killing the process.
  3. Restart the job with verbatim-studio process myfile.txt.

⚠️ If you’re running in a CI pipeline, ensure the job timeout is long enough.

3. General Troubleshooting¶

Area What to check How to check
Installation Correct package version verbatim-studio --version
Dependencies All required libraries are installed pip list | grep verbatim-studio
Configuration config.yaml syntax is valid yamllint config.yaml
Environment Variables like VERBATIM_TOKEN are set env | grep VERBATIM
Logs Look for detailed error messages verbatim-studio --log-level debug

3.1 Common “I can’t get it to start” problems¶

  • Missing dependencies – run pip install -r requirements.txt or npm install.
  • Permission denied on config file – chmod 600 ~/.verbatim-studio/config.yaml.
  • Port conflicts – if the tool uses a local server, ensure the port isn’t taken (lsof -i :<port>).
