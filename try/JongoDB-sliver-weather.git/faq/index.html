Frequently Asked Questions¶

Introduction to Sliver-Weather¶

What is sliver-weather.git?

Sliver-weather.git is a Docker Compose deployment designed for cybersecurity training and demonstration purposes. It showcases the Sliver C2 framework, utilizing a weather website as a cover application, nginx as a reverse proxy, and the Sliver C2 server for command and control operations.

Installation¶

How do I install sliver-weather.git?

To install sliver-weather.git, you'll need to have Docker and Docker Compose installed on your system. Clone the repository using git clone https://github.com/your-repo/sliver-weather.git, then navigate into the cloned directory. Run docker-compose up -d to start the containers in detached mode.

git clone https://github.com/your-repo/sliver-weather.git
cd sliver-weather
docker-compose up -d
What are the system requirements for running sliver-weather.git?

Sliver-weather.git requires a system with Docker and Docker Compose installed. Ensure your system meets the minimum requirements for Docker and has sufficient resources (CPU, memory, and disk space) to run the containers smoothly.

Basic Usage¶

How do I access the weather website?

Once the containers are up and running, you can access the weather website by navigating to http://localhost:8080 in your web browser. This will take you to the cover application, which is the weather website.

How do I interact with the Sliver C2 server?

To interact with the Sliver C2 server, you'll need to access the Sliver C2 web interface. This is typically done by navigating to http://localhost:8080/sliver in your web browser. You may need to authenticate using the credentials specified in the docker-compose.yml file or the Sliver C2 server configuration.

Configuration¶

How do I configure the Sliver C2 server?

The Sliver C2 server configuration can be modified by editing the sliver.yaml file in the repository. This file contains settings such as the server's listening address, port, and authentication credentials. After making changes, you'll need to restart the Sliver C2 container using docker-compose restart sliver.

# sliver.yaml example
server:
  addr: 0.0.0.0
  port: 8080
auth:
  username: admin
  password: password123
Can I customize the weather website?

Yes, the weather website can be customized by modifying the HTML, CSS, and JavaScript files in the weather-website directory. You can also add your own weather data API or modify the existing one to suit your needs.

Troubleshooting¶

What if the containers don't start?

If the containers don't start, check the Docker Compose logs using docker-compose logs to identify any errors. Common issues include insufficient resources, incorrect configuration, or conflicts with other containers.

How do I troubleshoot Sliver C2 server issues?

To troubleshoot Sliver C2 server issues, check the Sliver C2 server logs, which are typically located in the sliver-logs directory. You can also increase the log level by modifying the sliver.yaml file to get more detailed output.

Contributing¶

How can I contribute to sliver-weather.git?

To contribute to sliver-weather.git, fork the repository, make your changes, and submit a pull request. Ensure your changes are properly tested and follow the existing code style and conventions. You can also open issues to report bugs or suggest new features.

What are the guidelines for contributing code?

When contributing code, ensure you follow the existing code style and conventions. Use Markdown format for documentation, and include code examples where helpful. Test your changes thoroughly before submitting a pull request.
