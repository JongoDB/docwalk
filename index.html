Workflow

Init → Analyze → Deploy → Sync

Four steps from bare repo to live documentation. Every step is automated, every output is configurable.

01

Initialize

Point DocWalk at any Git repository — local, GitHub, GitLab, or Bitbucket. An interactive setup wizard walks you through language detection, doc structure preferences, deployment target, and domain configuration. Generates a docwalk.config.yml committed to your repo.

npx docwalk init --repo owner/repo
02

Analyze & Generate

Static analysis engine parses your entire codebase — AST extraction for function signatures, class hierarchies, type annotations, and module boundaries. Generates Markdown documentation organized into logical sections: Overview, API Reference, Architecture, Guides, and Changelog. Creates mkdocs.yml with auto-generated nav tree and search indices.

npx docwalk generate
03

Deploy

Choose your hosting provider and domain. DocWalk provisions the project, installs the CI/CD pipeline (GitHub Actions YAML, wrangler.toml, or vercel.json), configures DNS records for your custom domain, provisions SSL, and pushes the built site live. One command, zero manual steps.

npx docwalk deploy --provider cloudflare --domain docs.example.com/project
04

Smart Sync

On every push to the configured branch, DocWalk runs git diff against the last indexed commit. Only changed, renamed, or deleted files are re-analyzed. Cross-file impact analysis detects downstream documentation changes (e.g., a renamed export referenced elsewhere). Incremental build, incremental deploy — sub-30-second updates for typical PRs.

Runs automatically via CI/CD — or: npx docwalk sync

Configuration

One YAML. Full control.

Every aspect of DocWalk is configurable through a single docwalk.config.yml committed to your repo. Sensible defaults mean you can start with zero config.

docwalk.config.yml
# ─── Source Configuration ───
source:
  repo: JongoDB/CYROID
  branch: main
  include: [src/**, lib/**, api/**]
  exclude: [tests/**, node_modules/**]
  languages: auto

# ─── Analysis Engine ───
analysis:
  depth: full # full | surface | api-onlyExtracts every function, class, and type
  ai_summaries: true
  dependency_graph: true
  changelog: true

# ─── Sync Strategy ───
sync:
  trigger: on_push # on_push | cron | manualDocs update on every commit
  cron: "0 */6 * * *"
  diff_strategy: incrementalOnly re-analyzes changed files
  impact_analysis: true
docwalk.config.yml (continued)
# ─── Deployment ───
deploy:
  provider: cloudflare # gh-pages | cloudflare | vercel | netlify | s3
  project: cyroid-docs
  auto_ssl: true

# ─── Domain Routing ───
domain:
  custom: docs.fightingsmartcyber.com
  base_path: /cyroid
  dns_auto: true

# ─── Theme ───
theme:
  preset: developer # developer | corporate | startup | minimal | api-reference | knowledge-base | custom
  logo: ./assets/logo.svg
  favicon: ./assets/favicon.png

# ─── Hooks ───
hooks:
  pre_build: ["npm run lint:docs"]
  post_deploy: ["curl -X POST $SLACK_WEBHOOK"]
Smart Sync Engine

Change 3 files. Rebuild 3 pages.

DocWalk stores a commit-indexed analysis cache. On each sync, it diffs against the last indexed commit and only re-processes what changed — with cross-file impact detection.

Enterprise Specification

Technical deep-dive

Full product specification for technical evaluation and procurement.

CLI Runtime
Node.js 18+ (distributed via npm/npx). Zero global install required — npx docwalk works out of the box. Also available as a standalone binary via pkg for CI environments without Node.
Language Support
Supported now: TypeScript, JavaScript, Python, Go, Rust, Java, C#, Ruby, PHP (9 languages with full AST parsing via tree-sitter). Roadmap: Swift, Kotlin, Scala, Elixir, Dart, Lua, Zig, Haskell, C/C++. Plugin API planned for custom language parsers.
Analysis Engine
Multi-pass pipeline: (1) file-level AST parsing via tree-sitter, (2) symbol extraction (functions, classes, types, exports), (3) cross-file dependency resolution, (4) docstring/JSDoc/TSDoc harvesting, (5) optional LLM summarization (OpenAI, Anthropic, or local models). Outputs intermediate JSON manifest cached per commit SHA.
Git Diff Strategy
Compares current HEAD against last indexed commit SHA (stored in .docwalk/state.json). Uses git diff --name-status for file-level changes, then runs targeted re-analysis only on modified/added files. Cross-reference impact analysis detects downstream doc changes (e.g., a renamed export referenced in another module's docs).
Doc Generation
Produces standard MkDocs-compatible Markdown. Sections auto-generated: Project Overview, Getting Started, Architecture, API Reference (per module), Models/Types, Configuration, Changelog. Nav tree generated from directory structure with intelligent grouping. Full mkdocs.yml generated with Material theme config, plugins (search, minify, git-revision), and custom overrides.
Sync Triggers
on_push: GitHub Actions / GitLab CI trigger on push to configured branch. cron: Scheduled sync at user-defined intervals. manual: CLI command npx docwalk sync. webhook: HTTP endpoint for custom CI pipelines. All modes support dry-run (--dry-run) to preview changes without deploying.
Deploy Pipeline
GitHub Pages: Pushes to gh-pages branch via GitHub Actions. Generates .github/workflows/docwalk.yml. Cloudflare: Uses Wrangler CLI / Cloudflare API to create Pages project, upload assets, configure DNS. Vercel: Uses Vercel API to create project, deploy, and configure domain. Netlify: Uses Netlify CLI for deploy + _redirects config. S3: Direct upload with CloudFront invalidation support.
Web UI Dashboard Pro
Browser-based deployment management. View sync status across all repos, trigger manual rebuilds, manage custom domains, and monitor build logs — all from a single dashboard. No CLI required for day-to-day operations.
Domain Management
Auto-configures CNAME/A records via provider DNS API. Supports: subdomain routing (docs.example.com), subpath routing (example.com/docs), and root domain hosting. SSL provisioned automatically through provider (Let's Encrypt / Cloudflare). For subpath routing on non-Pages providers, generates reverse proxy configs (nginx, Caddy) or provider-specific rewrite rules.
Versioning
Mike-compatible versioning from git tags or branches. Each version is a separate doc build deployed to its own path (e.g., /v1.2/, /latest/). Version selector dropdown auto-generated in the Material theme. Configurable default version and alias support (e.g., /stable/ → /v2.1/).
Security
Secrets stored in CI environment variables — never in config files. GitHub App installation for repo access (no PATs). Cloudflare/Vercel tokens scoped to specific projects. Optional: deploy previews restricted to authenticated users via provider access policies. Enterprise: RBAC for who can trigger syncs and deploys.
Extensibility
Hook system with 6 lifecycle hooks: pre-analyze, post-analyze, pre-build, post-build, pre-deploy, post-deploy. Run shell commands at each stage (linting, notifications, custom transforms). Plugin API for custom analyzers planned for future release. MkDocs plugin ecosystem fully supported.
Monorepo Support
Analyzes multiple packages/services within a single repo. Each package can have its own doc section or be deployed as a separate site. Shared dependency docs are de-duplicated. Workspace detection for npm, yarn, pnpm, Go modules, and Cargo workspaces.
Performance
Full initial analysis: ~30s for 500-file repos, ~2min for 5000+ files (parallelized across CPU cores). Incremental sync: <5s for typical PR-sized changes. Built site: static HTML served from CDN edge — sub-50ms TTFB globally. Search index: client-side lunr.js, no server required.
