Features¶

A comprehensive guide to everything docwalk can do.

Commands¶

docwalk ci-setup¶

Generates provider-specific CI/CD pipeline configuration. Supports both production deploy and PR preview workflows.

docwalk deploy¶

Orchestrates: build → provider auth check → deploy → domain setup. Provides user-friendly errors when external tools are missing.

docwalk dev¶

Starts Zensical dev server for local preview. If Zensical isn't installed, auto-installs it into .docwalk/venv/ (no system pip needed).

docwalk doctor¶

Checks prerequisites and installs Zensical into a managed venv at .docwalk/venv/ — no system-wide pip install needed.

docwalk generate¶

Runs the full analysis → generation pipeline: 1. Load config (auto-init if missing) 2. Analyze codebase (with ora spinners) 3. Generate MkDocs Material site

docwalk init¶

Interactive setup wizard with two tracks: Quick Start — 3 steps: confirm repo, AI setup, theme → generate Custom — Full 6-step wizard (repo, analysis, deploy, domain, sync, theme)

docwalk status¶

Shows current sync state, project info, and config summary.

docwalk sync¶

Runs incremental sync: git-diff → targeted re-analysis → rebuild.

docwalk undeploy¶

Removes a deployed documentation site from the hosting provider.

docwalk version-list¶

List available versions from git tags matching the configured pattern.

docwalk version-deploy¶

Deploy a specific version tag using mike.

Options:

  • --tag
